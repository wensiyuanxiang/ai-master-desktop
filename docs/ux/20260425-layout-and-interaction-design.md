## 1. 全局布局框架

```
┌──────┬──────────────────────────────────────┬─────┐
│ 主导航 │            主内容区                    │ 右侧 │
│ 42px │      标题栏 + 右上角工具栏               │ 面板 │
│      │  ─────────────────────────────────── │360px│
│  💬   │                                      │ 按需 │
│      │         核心内容区                      │ 滑出 │
│  🔧   │    （随导航切换）                       │     │
│      │                                      │     │
│  📦   │                                      │     │
│      │                                      │     │
│  👤   │                                      │     │
│      │  ─────────────────────────────────── │     │
│      │         底部状态栏                      │     │
└──────┴──────────────────────────────────────┴─────┘
```

### 1.1 区域说明

| 区域 | 宽度 | 行为 |
|------|------|------|
| 左侧主导航 | 42px 固定 | 4 个图标入口，垂直排列，上方为功能，下方为设置 |
| 主内容区 | 自适应剩余 | 主体单屏，承载列表、对话、表单等核心内容 |
| 右侧滑出面板 | 360px | 默认隐藏，由右上角操作或列表项点击触发，从右侧滑入 |
| 底部状态栏 | 28px 固定 | 始终显示当前生效的套餐 + 模型 |

### 1.2 导航结构

```
顶部（功能入口）
  ├── 💬  对话       （默认首页）
  ├── 🔧  AI 工具     （管理编程工具配置）
  └── 📦  模型管理    （管理套餐库存）

底部（系统入口）
  └── 👤  个人配置    （通用设置）
```

主导航为图标 + 悬浮 tooltip 显示文字，选中态高亮（左侧边框或背景色变化）。

## 2. 各模块页面设计

### 2.1 对话（Chat）— 默认首页

**参考布局模式**：参考主流 AI 编程工具（Claude Code / Cursor）的对话布局 —— 模型/角色/工作目录选择与输入区一体，位于聊天窗口底部。

**全屏布局**：

```
┌──────┬──────────┬──────────────────────────────────┬─────┐
│ 主导航 │ 对话列表   │         对话内容区       [🧑‍🎓角色库]│ 右侧 │
│ 42px │ 240px    │                          [+新建]│360px│
│      │          │                                  │     │
│  💬   │ [+新建]  │                                  │     │
│      │ ──────── │    ┌──────────────────┐           │     │
│      │ 搜索...   │    │ 用户消息气泡       │           │     │
│      │          │    └──────────────────┘           │     │
│  🔧   │ 对话1    │                                  │     │
│      │ 对话2    │    ┌──────────────────┐           │     │
│      │ 对话3    │    │ AI 回复内容       │           │     │
│  📦   │          │    │ （流式输出）       │           │     │
│      │          │    └──────────────────┘           │     │
│      │          │                                  │     │
│  👤   │          │  ────────────────────────────     │     │
│      │          │  [输入框                     ]     │     │
│      │          │  ┌──────────────────────────┐    │     │
│      │          │  │ [👤 角色 ▼] [🤖 模型 ▼] [📂 目录]│    │ ← 输入工具栏
│      │          │  └──────────────────────────┘    │     │
└──────┴──────────┴──────────────────────────────────┴─────┘
│  🟢 当前: Claude Pro (Sonnet 4.6)   |  角色: Python 专家    │
└─────────────────────────────────────────────────────────────┘
```

**输入工具栏详解**（位于输入框下方，与输入区一体）：

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  [输入你的问题...                          ] [发送]  │
│                                                     │
│  ┌─────────────────────────────────────────────┐   │
│  │ [👤 Python专家 ▼]  [🤖 GPT-4o ▼]  [📂 ~/project] │   │
│  │  切换角色            切换模型       工作目录   │   │
│  │                                [浏览...]       │   │
│  └─────────────────────────────────────────────┘   │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**输入工具栏三要素**：

| 控件 | 功能 | 数据来源 |
|------|------|---------|
| 角色选择器 | 快捷切换当前对话使用的 AI 专家角色 | 角色库 |
| 模型选择器 | 切换当前对话使用的模型/套餐；下拉列表按厂商分组 | 模型管理的套餐库存 |
| 工作目录 | 设置当前对话的工作上下文目录，点击"浏览"打开系统目录选择器 | 本地文件系统 |

**对话历史区**（主屏内嵌左侧，240px）：
- 顶部 `[+ 新建对话]` 按钮
- 搜索框：按标题关键词过滤
- 对话列表：显示标题 + 最后活跃时间
- 右键菜单：重命名、删除
- 默认选中最近对话

**右上角标题栏**（对话页）：

| 控件 | 功能 |
|------|------|
| `[🧑‍🎓 角色库]` | 点击 → 右侧面板滑出 → 角色库全量管理（列表 + 搜索 + CRUD + 置顶 + 标签） |
| `[+ 新建]` | 新建空白对话 |

**右侧面板**（对话页触发场景）：

| 触发来源 | 面板内容 |
|---------|---------|
| 右上角"角色库"按钮 | 角色库全量管理（列表 + 搜索 + CRUD） |
| 对话历史某条目的右键 | 对话详情/重命名 |
| 无（默认隐藏） | — |

**角色选择器交互细节**：
- 下拉列表显示角色名称 + 简短描述
- 置顶角色在列表顶部（带图钉标识），与非置顶角色用分割线隔开
- 选中后立即切换 System Prompt，后续消息使用新角色
- 切换角色不中断当前对话

## 3. 全局元素

### 3.1 底部状态栏

```
┌──────────────────────────────────────────────────────────────┐
│  🟢 当前: Claude Pro (GPT-4o)  |  角色: Python 专家          │
└──────────────────────────────────────────────────────────────┘
```

- 始终可见，28px 高度
- 左侧显示生效的套餐和模型名
- 右侧显示当前角色（若在对话页）
- 状态指示灯：绿色 = 已配置生效，灰色 = 未配置

### 3.2 右侧面板通用行为

- 宽度 360px
- 从右侧滑入（transition 200ms ease-out）
- 遮罩层覆盖主内容区（半透明，点击可关闭面板）
- 面板内有独立滚动区域
- ESC 键关闭面板

### 3.3 通知/提示

- 操作反馈使用 toast 通知，位于右下角（状态栏上方）
- 类型：成功（绿色）、错误（红色）、警告（黄色）、信息（蓝色）
- 自动消失（3-5 秒），关键操作需手动关闭

## 5. 视觉规范

| 元素 | 规范 |
|------|------|
| 字体 | 系统默认等宽/无衬线，标题 16px，正文 14px，辅助 12px |
| 圆角 | 卡片 8px，按钮 6px，输入框 6px |
| 间距 | 模块间 24px，组件间 16px，组件内 12px |
| 颜色 | 深色模式为主，选中态用主题色高亮（蓝/紫系） |
| 图标 | Lucide React 图标集，导航图标 20x20px，功能图标 16x16px |
| 滚动条 | 细滚动条（6px），默认半透明 |

## 6. 关键交互流程

### 6.1 切换 Claude Code 套餐

```
对话/任意页面
  → 点左侧导航 [AI 工具]
  → 点击 [Claude Code] 卡片
  → 右侧面板滑出
  → 下拉选择目标套餐（数据来自模型管理）
  → 点 [应用配置]
  → Toast 提示"配置已生效"
  → 底部状态栏更新
```

### 6.2 管理角色并在对话中使用

**添加新角色**：
```
对话页面
  → 点击右上角 [🧑‍🎓 角色库]
  → 右侧面板滑出 → 显示完整角色库
  → 点击 [+ 添加角色]
  → 填写名称 / 描述 / System Prompt / 标签
  → 可选：勾选"置顶"
  → 保存
  → 角色列表刷新，新角色出现在对应位置
```

**在对话中切换角色**：
```
对话页面
  → 点击输入工具栏 [👤 角色 ▼] 下拉
  → 置顶角色在上方，其余按时间排列
  → 选中目标角色
  → 系统提示 "已切换至角色: Python专家"
  → 继续对话，后续消息使用新 System Prompt
```

### 6.3 添加新套餐

```
  → 点左侧导航 [模型管理]
  → 点击 [+ 添加套餐]
  → 右侧面板滑出编辑表单
  → 填写厂商/套餐/Key/URL/模型
  → 保存
  → 面板收起，列表刷新
  → 套餐即可在对话（模型选择）和 AI 工具（配置应用）中使用
```

## 7. MVP 范围确认

| 模块 | 范围 | 优先级 |
|------|------|------|
| 对话 | 文本对话 + 流式输出 + 对话历史列表（内嵌左侧） + 输入工具栏（角色/模型/工作目录切换） | P0 |
| 角色库 | 完整 CRUD + 置顶 + 标签分类 + 从对话输入工具栏"管理角色"入口触发右侧面板 | P1 |
| AI 工具 | Claude Code 配置管理（读写 `claude.json`） | P0 |
| 模型管理 | 套餐完整 CRUD + 厂商分组 | P0 |
| 个人配置 | 语言切换 + 数据导出/导入 | P1 |
| 备份管理 | 自动备份 + 列表 + 恢复 + 导出 | P1 |
| 安全 | API Key 加密存储、前端不可见明文 | P0 |
| i18n | 中英文切换 | P1 |

# AI Master Desktop - 产品需求文档

生成日期：2026-04-25 | 目的：定义 AI 管家桌面应用 MVP 的功能边界与核心体验

---

## 1. 产品定位

AI Master Desktop 是一款跨平台桌面应用，定位为用户的"AI 管家"——统一管理各类 AI 编程工具的订阅套餐、API 配置、AI 专家角色库，并内置通用咨询对话能力。用户无需手动编辑 JSON 配置文件即可一键切换不同厂商套餐与模型配置。

---

## 2. 目标用户与场景

### 2.1 目标用户
- 个人用户，非团队协作
- 同时使用多个 AI 编程工具（Claude Code、Gemini、国内大厂 GLM 等）
- 拥有多个账号/套餐，需要频繁切换

### 2.2 核心场景
1. **套餐切换**：持有多个 Claude Code 套餐，在不同项目或用量阶段需要切换 API Key / Base URL / 模型，通过桌面一键完成
2. **角色切换**：日常有编程指导、文章写作、翻译、头脑风暴等多类需求，需在不同 AI 专家角色间快速切换并开始对话
3. **通用咨询**：将管家作为统一 AI 对话入口，选择套餐与角色后直接发起咨询

## 3. 功能需求（MVP）

### 3.1 套餐管理

**描述**：管理各厂商的 API 订阅套餐信息。

| 需求项 | 说明 |
|-------|------|
| 添加套餐 | 输入厂商名称（如 Claude Code / Gemini / GLM）、套餐名称、API Key、Base URL、模型名，可设置订阅周期（起始/到期日） |
| 编辑套餐 | 修改已保存的套餐信息 |
| 删除套餐 | 删除套餐，需二次确认 |
| 套餐列表 | 展示所有已添加套餐，区分厂商标签，支持搜索和筛选 |
| 切换套餐 | 选中某套餐后一键应用为当前激活配置 |

**优先级**：P0（MVP 核心）

### 3.2 配置切换（Claude Code 优先）

**描述**：管理并切换 `~/.claude/claude.json` 配置文件，支持 Windows / Linux / macOS。

| 需求项 | 说明 |
|-------|------|
| 部分编辑模式 | 仅替换 `claude.json` 中的 API Key（或 Key + Base URL + Model），保留其他字段不变 |
| 完整编辑模式 | 直接编辑完整的 `claude.json` 文件内容，提供 JSON 编辑器 |
| 配置备份 | 切换前自动备份当前配置，可按套餐名+时间戳命名备份快照 |
| 备份恢复 | 浏览历史备份列表，选择某个备份恢复 |
| 备份导出/导入 | 支持将备份文件导出到指定目录，或从外部导入备份文件 |
| 路径自适应 | 自动检测当前操作系统 `~/.claude/claude.json` 路径，支持手动指定路径 |
| 配置预览 | 编辑或切换时可预览 `claude.json` 的当前有效内容 |

**优先级**：P0（MVP 核心）

### 3.3 AI 专家角色库

**描述**：用户自定义的 AI 专家角色集合，每个角色包含 System Prompt 模板。

| 需求项 | 说明 |
|-------|------|
| 添加角色 | 输入角色名称、描述、System Prompt 正文、标签（可选，用于分类） |
| 编辑角色 | 修改已有角色信息 |
| 删除角色 | 删除角色，需二次确认 |
| 角色列表 | 展示所有角色，支持按标签筛选和关键词搜索 |
| 角色置顶 | 常用角色可置顶显示 |
| 选择角色 | 在咨询对话中选定角色作为对话的 System Prompt |
| 角色导入/导出 | 支持将角色导出为 JSON 文件，或从外部导入 |

**优先级**：P1（MVP 重要组成）

### 3.4 通用咨询对话

**描述**：在应用内与选定的 AI 模型进行对话，消耗用户管理的 API Key。

| 需求项 | 说明 |
|-------|------|
| 对话创建 | 选择套餐（API）+ 选择角色（System Prompt）后新建对话 |
| 流式输出 | 对话过程支持流式响应，实时显示回复内容 |
| 多轮对话 | 保持上下文，支持连续多轮 |
| 对话历史 | 所有对话持久化存储到本地，支持查看历史对话列表 |
| 继续对话 | 可打开历史对话并继续交流 |
| 删除对话 | 删除不需要的对话记录 |
| 对话重命名 | 支持手动重命名对话标题 |
| 切换角色/模型 | 对话过程中可切换当前使用的角色或套餐 |

**优先级**：P1（MVP 重要组成）

### 3.5 配置备份管理

**描述**：本机本地备份，保障配置安全可恢复。

| 需求项 | 说明 |
|-------|------|
| 自动备份 | 切换配置时自动生成备份快照 |
| 备份列表 | 查看所有备份，显示套餐名、时间戳 |
| 备份恢复 | 选择备份恢复至 `claude.json` |
| 备份删除 | 删除过期或不需要的备份 |
| 备份导出 | 将备份文件导出到指定本地路径 |

**优先级**：P1

## 4. 非功能需求

| 类别 | 说明 |
|-----|------|
| 平台 | Windows / Linux / macOS，UI 与交互保持一致 |
| 安全 | API Key 等敏感信息仅存储在本地，由 Rust 侧加密读写，前端不持有密钥明文 |
| 国际化 | 所有用户可见文案通过 i18n 配置，默认支持中文、英文，预留扩展 |
| 离线可用 | 除咨询对话外，套餐管理、角色管理、配置管理均不依赖网络 |
| 性能 | 应用启动 < 3 秒，对话流式响应延迟感不明显 |
| 日志 | Rust 侧使用 tracing，不记录密钥/Token 等敏感信息 |

## 6. 数据模型概要

### 6.1 Provider / Subscription
```
Provider (厂商)
  - id, name, created_at
  - Subscription (套餐)
    - id, provider_id, name, api_key, base_url, model, 
      start_date, end_date, is_active, created_at, updated_at
```

### 6.2 Expert Role
```
Role (AI 专家角色)
  - id, name, description, system_prompt, tags, is_pinned, 
    created_at, updated_at
```

### 6.3 Conversation
```
Conversation (对话)
  - id, title, subscription_id, role_id, created_at, updated_at

Message (消息)
  - id, conversation_id, role (user/assistant), content, created_at
```

### 6.4 Config Backup
```
ConfigBackup (配置备份)
  - id, subscription_name, file_path, checksum, 
    created_at, note
```

## 7. 验收标准

- [ ] 可添加、编辑、删除、切换厂商套餐
- [ ] 可一键将选中套餐的配置写入 `claude.json`（支持部分编辑与完整覆盖两种模式）
- [ ] 切换配置时自动备份当前配置，可恢复历史备份
- [ ] 可添加、编辑、删除、搜索、置顶 AI 专家角色
- [ ] 选择套餐 + 角色后可发起对话，支持流式输出与多轮
- [ ] 对话历史持久化，可继续、删除、重命名
- [ ] 所有用户可见文案走 i18n
- [ ] 应用在 Windows / macOS / Linux 上行为一致
- [ ] API Key 不出现在前端日志或本地存储明文
